# CURSOR SYSTEM PROMPT

Проект: AI Task Assistant / Time Management

Перед началом любой задачи обязательно изучи:

* docs/TZ_MVP.md
* docs/ARCHITECTURE_BASELINE.md
* docs/DATA_MODEL.md
* docs/API_CONTRACTS.md
* docs/ACCESS_CONTROL.md
* docs/AI_CONTRACTS.md
* docs/ADR/*

## Главная цель

Разработать self-hosted web-first систему управления задачами для личного, семейного и рабочего использования.

Система должна обеспечивать:

* личные задачи;
* семейные задачи;
* рабочие задачи;
* партнерские задачи;
* AI-классификацию;
* матрицу Эйзенхауэра;
* утренний дашборд;
* вечерний отчет;
* аналитику;
* напоминания;
* повторяющиеся задачи;
* строгую приватность данных.

---

## Архитектурные принципы

Следовать:

1. Modular Monolith.
2. Domain-driven design light.
3. Contract-first.
4. Security-first.
5. Privacy-first.
6. Event-driven analytics.
7. AI как отдельный слой.

---

## Запрещено

Не создавать:

* микросервисы;
* Kubernetes;
* сложную распределенную архитектуру;
* собственный auth framework;
* собственную очередь сообщений;
* собственную ORM;
* собственный state manager без необходимости.

---

## Разделение слоев

Обязательно разделять:

UI Layer
API Layer
Domain Layer
Access Control Layer
Persistence Layer
AI Layer
Worker Layer
Notification Layer

---

## Требования к коду

1. TypeScript strict mode.
2. Полная типизация.
3. Отсутствие any без обоснования.
4. Явные DTO.
5. Явные domain models.
6. Чистые интерфейсы.
7. Никакой бизнес-логики в React компонентах.
8. Никакой логики доступа в UI.

---

## Работа с задачами

Каждая задача должна:

* принадлежать пространству;
* иметь владельца;
* иметь статус;
* иметь события в TaskEvent.

Запрещено создавать задачи без event logging.

---

## Работа с доступом

Все проверки доступа выполнять на backend.

Нельзя полагаться на frontend ограничения.

---

## Работа с AI

AI не имеет права:

* изменять доступы;
* удалять задачи;
* назначать пользователей без подтверждения.

AI только предлагает.

Окончательное решение принимает пользователь.

---

## Перед завершением задачи

Всегда предоставляй:

1. список измененных файлов;
2. описание изменений;
3. риски;
4. список тестов;
5. что осталось сделать.

Никогда не завершай задачу без acceptance criteria.
